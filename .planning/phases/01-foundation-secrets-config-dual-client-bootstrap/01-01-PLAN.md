## Phase Goal

**As an** operator of Mr. Kek, **I want to** run one process that loads and validates every secret and chat id from the environment and knows the hardcoded participant identities, **so that** the bot can never boot half-configured or leak a credential from source.

<objective>
Scaffold the greenfield TypeScript ESM project and deliver the two inputs every later phase consumes: a fail-fast env config loader and the hardcoded participant/identity constants. This is the foundation slice of the walking skeleton — after this plan the project compiles, the config loader proves it rejects an incomplete environment by name, and the constants module exposes the exact identities from SPEC §6.

Purpose: Lock the secret↔constant boundary (D-03) and guarantee the process refuses to start when misconfigured (D-02, Success Criterion 1) before any Telegram client exists. No secret may appear in source (CFG-01, Success Criterion 2).

Output: package.json (ESM, locked stack, npm scripts), tsconfig.json (NodeNext strict), .gitignore, .env.example (authoritative key list), src/config/env.ts (loadConfig + Config type), src/config/constants.ts (participants, admin, MR_KEK_ID, triggers), src/logger.ts (pino with redaction).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process launcher (Doppler/dotenv) -> app | Secrets cross from the secret store / .env into process.env at launch; the app must treat them as the only source and never echo them |
| source tree -> git history | A committed secret (esp. the leaked StringSession) is a permanent account-takeover credential |
| app -> logs | Loaded secrets must not be re-emitted via logging |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env / .env.example / source | mitigate | .gitignore blocks all .env files; .env.example carries placeholders only (no real values, no leaked session); grep gate in acceptance criteria asserts no secret literal in src/ (CFG-01) |
| T-01-02 | Information Disclosure | src/logger.ts | mitigate | pino redact for SESSION_KEY/BOT_TOKEN/token/session fields; never log raw config object (Security Domain V7) |
