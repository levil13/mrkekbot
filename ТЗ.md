# ТЗ — Telegram бот «Мистер Кек»

## Общее описание
Telegram-бот для геймификации групповых чатов. Участники зарабатывают и тратят «кеки» — внутреннюю валюту чата.

---

## Стек
- **Runtime:** Node.js + TypeScript
- **Bot API:** Telegraf
- **User API:** GramJS (telegram)
- **БД:** lowdb (JSON-файл `db.json`)
- **Процесс:** PM2

---

## Функциональные требования

### 1. Система кеков
- Пользователь пишет `кек` / `kek` / `топкек` / `topkek` / `k3k` — отдаёт 1 кек автору предыдущего сообщения (или сообщения на которое сделан реплай)
- Нельзя ставить кек самому себе
- Пишет `некек` / `nekek` — отзывает последний поставленный кек
- Если 3 разных пользователя поставили кек одному сообщению — бот объявляет «тройной кек»
- При постановке кека боту — кек перенаправляется Лукасу

### 2. Кеказино
- **Триггер:** `/kekcasino` или текстовые ключи (`кеказино`, `кек казино`, `рандомный кек`, `kekasino`, `kek casino`)
- **Стоимость:** 1 кек списывается только после успешной отправки мема
- Бот берёт случайный мем (фото/видео) из канала `ANIME_KONFA_ID` и пересылает в чат
- При балансе 0 — отказ
- **Награда за хороший мем:** если другой участник ставит `кек` на мем из казино — игрок, который его выиграл, получает +1 кек. Для этого бот хранит соответствие `{ messageId → playerId }` в БД

### 3. Бан медиа (`#кал` / `#kal`)
- Реплай на казино-мем с текстом `#кал` или `#kal` — добавляет медиа в бан-лист
- Забаненные медиа не попадают в казино
- Бан-лист сохраняется в БД и **не сбрасывается** при `/reset`
- Повторный бан одного мема игнорируется

### 4. Команды
| Команда | Доступ | Описание |
|---|---|---|
| `/start` | Только Лукас | Приветствие + статистика |
| `/stats` | Все | Таблица лидеров |
| `/keys` | Все | Список триггерных слов |
| `/reset` | Только Лукас | Сброс статистики (бан-лист сохраняется) |
| `/kekcasino` | Все | Запуск казино |
| `/help`, `/commands` | Все | Список команд |

## Нефункциональные требования
- **Отказоустойчивость:** глобальный `bot.catch()` — при ошибке бот не падает, пользователь получает сообщение
- **Мониторинг:** health check endpoint на порту `3002`, Uptime Kuma на порту `3001`
- **Конфигурация:** все чувствительные данные в `.env` (`BOT_TOKEN`, `API_ID`, `API_HASH`, `SESSION_KEY`, `ANIME_KONFA_ID`, `MR_KEK_ID`)
- **Запуск:** PM2, `npm run dev` (ts-node) для разработки, `npm run start` (собранный dist) для продакшна
